Open-source ERP platform for developers and business analysts

MyCompany is an open-source ERP built on lsFusion, a low-code ERP platform designed for people who write logic for a living. Instead of scattering business rules across stored procedures, backend services, and frontend code, you describe the data model, logic, and forms in a single declarative language — and the platform turns that description into a working three-tier system.

For custom ERP development this changes the economics. The application is plain text files under version control, the tooling is a fully-featured IDE based on IntelliJ IDEA, and your customizations live in separate modules that survive product updates. The platform is LGPL v3, the MyCompany application is Apache 2.0 — no license keys, no DRM, no lock-in.

One declarative language for the whole application

lsFusion is a declarative, function-level, reactive, event-driven language — its GitHub README describes it as a fifth-generation language. The same language defines tables, calculated properties, constraints, events, actions, and forms, so there is no impedance mismatch between an ORM, a service layer, and a UI framework. Everything is stored as plain text — "everything as code" — which means ordinary Git diffs, code review, and branching work exactly as they do in any software project.

Real developer tooling, not a form painter

Development happens in a fully-featured IDE built on IntelliJ IDEA: usages search, error highlighting, smart code completion, and a debugger that supports data-change breakpoints — pause execution the moment a specific value changes, wherever the change comes from. If you want to feel the language before installing anything, you can try lsFusion online.

Extend the ERP without forking it

Custom work goes into your own module: a .lsf file that starts with MODULE and REQUIRE MyCompany. Drop it on the server and point the logics.topModule setting at it — no forking the product, no recompilation. From your module you extend or override the standard data model, forms, and logic. Updating the product means replacing a single JAR file, and your custom modules keep working on top of it.

Java, PostgreSQL, and ACID by default

The architecture is a classic three-tier design: an lsFusion application server on the JVM, a PostgreSQL database, and a web client that renders forms natively with CSS layout. Declarative logic is executed on the database server, and every operation is ACID out of the box — you never write manual locks or transaction handling. According to the platform vendor, it scales to thousands of simultaneous users and terabytes of data, with horizontal scaling of the application tier, and works well over low-bandwidth channels.

Integration through files and an HTTP API

MyCompany exchanges data in JSON, XML, CSV, XLS, and DBF for both export and import. Stock data is also exposed over an HTTP API in JSON — for example, the Inventory.getInventory endpoint — so external systems can read inventory without touching the database directly.

FAQ

What stack is it built on?

Java and PostgreSQL. The lsFusion application server runs on the JVM, the data lives in a standard PostgreSQL database, and users work through a web client. All application logic is written in the lsFusion language.

How do I set up a development environment?

Install the IntelliJ IDEA-based IDE with the lsFusion platform and follow the development guide linked above. You can also install MyCompany itself on your own server in 10-15 minutes and develop against a real instance.

How do custom modules survive product updates?

Your code lives in separate .lsf modules that require MyCompany rather than patch it. A product update replaces one JAR file; your modules stay on the server and continue to extend the new version — nothing is recompiled.

Is there an API?

Yes. Data can be exported and imported as JSON, XML, CSV, XLS, and DBF, and stock data is available over an HTTP API in JSON, such as Inventory.getInventory.

Can I ship commercial builds?

Yes. The MyCompany application is Apache 2.0 and the lsFusion platform is LGPL v3. Both are free for commercial use, with no license keys or DRM anywhere in the stack.

Where is the community?

There are Telegram and Slack channels (links in the footer), and documentation is available in English, Russian, and Polish. A live demo lets you explore a running system before writing any code.
