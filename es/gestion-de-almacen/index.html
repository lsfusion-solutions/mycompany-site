Gestión de almacén e inventario open source

MyCompany es un software de gestión de inventario gratuito y open source: hace el seguimiento de las existencias en cualquier número de almacenes, registra cada recepción, expedición y transferencia y muestra exactamente qué hay en existencias, reservado y disponible en cada momento. El código está publicado bajo la licencia Apache 2.0, así que no hay costes ocultos ni dependencia de licencias.

A diferencia de una aplicación aislada de control de existencias, MyCompany es un ERP completo para pequeñas empresas construido sobre la plataforma lsFusion (LGPL v3, Java y PostgreSQL, transacciones ACID). Su software de gestión de almacén comparte una única base de datos con las compras y las ventas, y puede instalarlo en su propio servidor en 10-15 minutos.

Almacenes, zonas y posiciones

Las ubicaciones forman un árbol jerárquico de profundidad arbitraria: el almacén, zonas dentro de él, posiciones (bins) dentro de las zonas. Puede gestionar cualquier número de almacenes y la estructura es flexible: el detalle a nivel de posición es opcional, así que un pequeño depósito puede ser una única ubicación, mientras que unas instalaciones mayores se desglosan con el detalle que necesite.

Cada movimiento de stock es un documento

Todos los cambios en el inventario pasan por documentos, lo que mantiene el control de existencias auditable:

  • Recepciones para las entradas de mercancía y expediciones para las salidas
  • Transferencias entre almacenes o posiciones
  • Ajustes de inventario: recuentos físicos que registran excedentes y faltantes
  • Bajas por daño, pérdida, defecto o por caducidad

Picking con códigos de barras en dispositivos móviles

Cuando una expedición marcada para picking alcanza el estado Listo, se genera automáticamente una tarea de picking. Cada línea registra la cantidad solicitada y la recogida, el operario y las marcas de tiempo. El personal del almacén trabaja desde un formulario de picking móvil, escaneando los códigos de barras de las posiciones y de los artículos, y también puede mover mercancía entre ubicaciones con transferencias móviles.

Trazabilidad por lotes y números de serie

Los lotes ofrecen trazabilidad por partidas en recepciones, expediciones, transferencias, bajas y ajustes. Los identificadores se generan automáticamente a partir de un prefijo y un numerador (por ejemplo, LOT-000123), y un modo de números de serie crea un lote por unidad cuando necesita trazabilidad a nivel de artículo. Al escanear un código de barras se obtiene el lote correspondiente, y un código desconocido puede crear uno nuevo al momento. Las etiquetas de código de barras de los lotes se imprimen en PDF, DOCX, XLSX, RTF o HTML.

Informes basados en registros fiables

Todas las cifras provienen de tres registros (inventario, reservas y coste), de modo que cualquier saldo puede rastrearse hasta los movimientos que lo originaron:

  • Informe de inventario: cantidades Esperada, Reservada, Disponible y En existencias a cualquier fecha, con filtros y vista pivotante
  • Valoración del inventario: coste unitario y total de las existencias
  • Movimientos de artículos: el historial completo de movimientos de un artículo
  • Informe de costes: cómo fluyen los costes por su inventario

Parte de un ERP completo, abierto a otros sistemas

Como el inventario vive dentro del ERP, los pedidos de compra crean automáticamente recepciones y las facturas de venta crean automáticamente expediciones, sin volver a teclear datos. Varios artículos de venta pueden corresponder a una única unidad de stock mediante un coeficiente de conversión, y las unidades de embalaje pueden definirse por tipo de documento. Las existencias actuales también están disponibles para sistemas externos a través de una API HTTP que devuelve JSON y admite filtros por categoría y ubicación.

FAQ

¿El software de inventario es realmente gratis?

Sí. MyCompany es open source bajo la licencia Apache 2.0, sin costes ocultos ni dependencia de licencias. Lo aloja usted mismo y la instalación en su propio servidor lleva 10-15 minutos.

¿Cuántos almacenes y ubicaciones puedo gestionar?

Cualquier número de almacenes. Cada uno es un árbol jerárquico de profundidad arbitraria (almacén, zonas, posiciones) y el seguimiento a nivel de posición es opcional, así que define solo la estructura que su operativa necesita.

¿Admite códigos de barras?

Sí. Los operarios escanean los códigos de las posiciones y de los artículos en el formulario de picking móvil, un código escaneado se resuelve a su lote (los códigos desconocidos pueden crear uno) y las etiquetas de código de barras de los lotes se imprimen en PDF, DOCX, XLSX, RTF o HTML.

¿Puedo hacer trazabilidad de lotes y números de serie?

Sí. La trazabilidad por partidas funciona en recepciones, expediciones, transferencias, bajas y ajustes, con identificadores de lote generados automáticamente. Un modo de números de serie asigna un lote por unidad para la trazabilidad a nivel de artículo.

¿Funciona en dispositivos móviles?

Sí. El personal del almacén usa un formulario de picking móvil con escaneo de códigos de barras y puede realizar transferencias entre ubicaciones desde un dispositivo móvil.

¿Pueden otros sistemas consultar mis existencias?

Sí. Las existencias actuales se exponen a través de una API HTTP que devuelve JSON y puede filtrarse por categoría y ubicación, de modo que los sistemas externos pueden consultar la disponibilidad directamente.
