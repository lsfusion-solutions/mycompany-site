面向开发者的 ERP · 一切皆代码

为开发者打造的开源 ERP 平台

MyCompany 是基于 lsFusion 的生产级 ERP，面向需要修改真实业务逻辑的团队。用一种声明式语言定义数据模型、计算、约束、操作与表单，把每次改动保存在 Git 中，并通过独立模块扩展产品而无需维护分叉。

专注业务系统，而不是框架样板代码

从数据库到界面使用同一套模型

lsFusion 将 ERP 的关键部分放在一致的代码库中。平台根据声明式定义生成持久化、事务和界面行为，而应用仍是普通文本，可使用标准的评审、分支与部署流程。

LSF

一种声明式语言

用同一种语言描述存储与计算数据、约束、事件、操作和表单，无需将规则分散到 ORM、服务与前端。

IDE

真正的开发工具

在基于 IntelliJ IDEA 的环境中使用代码补全、错误提示、引用搜索，以及数据变化断点调试。

MOD

用模块而不是分叉

将企业逻辑放在独立 .lsf 模块中并 REQUIRE MyCompany。升级只替换产品 JAR，扩展代码仍然保留。

清晰可理解的技术栈

从 .lsf 源码到三层 ERP

lsFusion 服务器运行在 JVM 上，使用 PostgreSQL 存储数据并提供原生 Web 客户端。声明式逻辑靠近数据执行，所有操作默认具备事务性，一个模块即可扩展所有层。

开发者常见问题

打开 IDE 之前

阅读开发指南

使用什么技术栈？

应用服务器运行在 JVM 上，数据存储在 PostgreSQL 中，用户通过 Web 客户端工作。应用逻辑与表单使用 lsFusion 编写。

如何搭建开发环境？

安装带 lsFusion 插件的 IntelliJ IDEA，克隆 MyCompany 仓库，并按照指南在本地运行应用和模块。

模块如何跨版本升级？

您的 .lsf 模块依赖 MyCompany，而不是修改产品。升级只替换产品 JAR，独立模块继续保留。

是否提供 API？

是。MyCompany 支持 HTTP、JSON、XML、CSV、XLS 与 DBF，外部系统无需直接访问数据库。

可以开发商业解决方案吗？

可以。MyCompany 使用 Apache 2.0，平台使用 LGPL v3；技术栈中没有许可证密钥或 DRM。

不安装能否试用语言？

可以。使用 lsFusion 在线环境、查看 GitHub 仓库，或在自己的服务器上安装 MyCompany。
