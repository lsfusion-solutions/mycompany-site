Servidor próprio · A sua infraestrutura

ERP self-hosted que corre no seu próprio servidor

O MyCompany é um sistema ERP gratuito e open source concebido para ser alojado por si. Instala-o no seu próprio servidor — físico ou virtual, no seu escritório ou na sua própria conta de cloud — e tudo fica lá: a aplicação, a base de dados e todos os registos que o seu negócio cria. Não há conta SaaS, não há subscrição por utilizador e não há um fornecedor entre si e os seus dados.

O ERP funciona onde você decide

Se procura um ERP self-hosted ou um ERP on-premise que uma equipa pequena consiga pôr de pé sem um projeto de consultoria, o MyCompany é uma escolha prática: instala-se no seu próprio servidor em 10–15 minutos, serve um cliente web de origem e é disponibilizado sob a licença Apache 2.0 — sem custos ocultos e sem dependência de licenças.

Operações sem pontos cegos

Saiba exatamente o que fica nas suas mãos

Atualizar um sistema self-hosted não deve ser um projeto em si mesmo. No MyCompany, uma atualização significa substituir um único ficheiro JAR no classpath do servidor — as versões publicadas estão em download.lsfusion.org. E como a aplicação é construída a partir de módulos lsFusion, pode estendê-la no seu próprio servidor sem recompilar nada: coloque um ficheiro .lsf junto da aplicação e aponte logics.topModule para ele.

Quais são os requisitos do servidor?

Uma máquina com pelo menos 4 GB de RAM a correr Ubuntu 18+, Debian 9+, RHEL 8+, CentOS 8+, Fedora 35+ ou Windows. O cliente web é servido na porta 8080 e o cliente desktop na porta 7652.

Quanto tempo demora a instalação?

Cerca de 10–15 minutos no seu próprio servidor. No Linux é um único comando; no Windows executa um instalador autónomo. Depois, basta abrir o cliente web no endereço do seu servidor, na porta 8080.

Posso executá-lo em Docker?

Sim. A implementação em Docker está oficialmente documentada, pelo que pode correr o MyCompany ao lado dos seus outros serviços em contentores.

Como funcionam as atualizações?

Substitui um único ficheiro JAR no classpath do servidor por uma versão mais recente de download.lsfusion.org. Sem reativação de licenças, porque não há licenças para ativar.

Funciona numa rede local, sem cloud?

Sim. Tudo corre no seu próprio servidor, pelo que o sistema funciona na sua LAN sem dependência de um serviço cloud externo — os colaboradores abrem simplesmente o cliente web no navegador.

Os meus dados ficam bloqueados?

Não. Os seus dados vivem numa base de dados PostgreSQL padrão no seu próprio servidor, e o código da aplicação é open source sob Apache 2.0. Pode consultar, copiar ou migrar os dados diretamente a qualquer momento.
