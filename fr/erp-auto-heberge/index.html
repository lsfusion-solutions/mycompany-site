Un ERP auto-hébergé qui tourne sur votre propre serveur

MyCompany est un système ERP gratuit et open source conçu pour l'auto-hébergement. Vous l'installez sur votre propre serveur — physique ou virtuel, dans vos locaux ou dans votre propre compte cloud — et tout y reste : l'application, la base de données et chaque enregistrement créé par votre entreprise. Pas de compte SaaS, pas d'abonnement par utilisateur, pas de fournisseur entre vous et vos données.

Si vous cherchez un ERP auto-hébergé ou un ERP on-premise qu'une petite équipe peut mettre en place sans projet de conseil, MyCompany est un choix pragmatique : il s'installe sur votre propre serveur en 10-15 minutes, sert un client web d'origine et est publié sous licence Apache 2.0 — sans coûts cachés ni dépendance aux licences.

Des opérations sans angle mort

Sachez exactement ce qui reste entre vos mains

Mettre à jour un système auto-hébergé ne devrait pas être un projet en soi. Dans MyCompany, une mise à jour consiste à remplacer un seul fichier JAR dans le classpath du serveur — les versions publiées sont sur download.lsfusion.org. Et comme l'application est construite en modules lsFusion, vous pouvez l'étendre sur votre propre serveur sans rien recompiler : déposez un fichier .lsf à côté de l'application et pointez logics.topModule vers lui.

Questions et réponses

Avant l'hébergement

Contactez-nous

Quelles sont les exigences serveur ?

Une machine avec au moins 4 Go de RAM sous Ubuntu 18+, Debian 9+, RHEL 8+, CentOS 8+, Fedora 35+ ou Windows. Le client web est servi sur le port 8080 et le client de bureau sur le port 7652.

Combien de temps prend l'installation ?

Environ 10-15 minutes sur votre propre serveur. Sous Linux, c'est une seule commande ; sous Windows, vous lancez un installateur autonome. Ensuite, il suffit d'ouvrir le client web à l'adresse de votre serveur, sur le port 8080.

Puis-je l'exécuter dans Docker ?

Oui. Le déploiement Docker est officiellement documenté : vous pouvez faire tourner MyCompany aux côtés de vos autres services conteneurisés.

Comment fonctionnent les mises à jour ?

Vous remplacez un seul fichier JAR dans le classpath du serveur par une version plus récente depuis download.lsfusion.org. Pas de réactivation de licence, puisqu'il n'y a aucune licence à activer.

Fonctionne-t-il sur un réseau local, sans cloud ?

Oui. Tout tourne sur votre propre serveur : le système fonctionne sur votre LAN sans dépendre d'un service cloud externe — les employés ouvrent simplement le client web dans leur navigateur.

Mes données sont-elles verrouillées ?

Non. Vos données vivent dans une base PostgreSQL standard sur votre propre serveur, et le code de l'application est open source sous Apache 2.0. Vous pouvez interroger, sauvegarder ou migrer les données directement à tout moment.
