Ein selbst gehostetes ERP, das auf Ihrem eigenen Server läuft

MyCompany ist ein kostenloses Open-Source-ERP-System, das für das Selbsthosten konzipiert ist. Sie installieren es auf Ihrem eigenen Server — physisch oder virtuell, im Büro oder in Ihrem eigenen Cloud-Konto — und alles bleibt dort: die Anwendung, die Datenbank und jeder Datensatz, den Ihr Unternehmen erzeugt. Kein SaaS-Konto, kein Abo pro Benutzer, kein Anbieter zwischen Ihnen und Ihren Daten.

Wenn Sie ein selbst gehostetes ERP oder ein On-Premise-ERP suchen, das ein kleines Team ohne Beratungsprojekt aufsetzen kann, ist MyCompany eine pragmatische Wahl: Es ist in 10-15 Minuten auf dem eigenen Server installiert, liefert einen Web-Client ab Werk und erscheint unter der Apache-2.0-Lizenz — ohne versteckte Kosten und ohne Lizenzbindung.

Installation: eine Zeile unter Linux, ein Installer unter Windows

Unter Ubuntu 18+ oder Debian 9+ besteht die gesamte Installation aus einem einzigen Befehl auf Ihrem Server:

source <(curl -s https://download.lsfusion.org/solutions/install-mycompany-ubuntu18.sh)

Eine CentOS-Variante des Skripts deckt RHEL 8+, CentOS 8+ und Fedora 35+ ab. Unter Windows gibt es einen eigenständigen Installer, und das Docker-Deployment ist offiziell dokumentiert, falls Sie Container bevorzugen. Nach Abschluss der Installation öffnen Sie http://ihre-server-ip:8080 im Browser und melden sich als admin mit leerem Passwort an — und ändern es natürlich.

Bescheidene Serveranforderungen

  • Mindestens 4 GB RAM — ein kleiner VPS oder ein Bürorechner reicht für den Anfang
  • Web-Client ab Werk auf Port 8080
  • Desktop-Client auf Port 7652 für alle, die ihn bevorzugen
  • Läuft problemlos im lokalen Netzwerk — die Mitarbeiter verbinden sich einfach per Browser

Ein Stack, den Sie von Anfang bis Ende kontrollieren

Unter der Haube ist MyCompany ein in Java geschriebener lsFusion-Anwendungsserver auf PostgreSQL, mit vollständigen ACID-Garantien für Ihre Transaktionen. Die lsFusion-Plattform steht unter LGPL v3, die MyCompany-Anwendung unter Apache 2.0. Das bedeutet: keine Lizenzschlüssel, kein DRM und keine Lizenzprüfung, die „nach Hause telefoniert“ — die installierte Software gehört ganz Ihnen.

Warum Unternehmen sich für das Selbsthosten entscheiden

  • Ihre Daten bleiben bei Ihnen. Kunden, Preise, Bestände und Finanzen liegen auf Ihrer Infrastruktur, nicht in der Cloud eines anderen.
  • Kein Abo pro Benutzer. Fügen Sie so viele Mitarbeiter hinzu, wie Sie brauchen, ohne dass die Rechnung wächst.
  • Läuft in Ihrem LAN. Der Web-Client läuft in Ihrem eigenen Netzwerk, nach Ihren Regeln.
  • Keine Anbieterbindung. Offener Quellcode und eine Standard-PostgreSQL-Datenbank: Sie sitzen nie in der Falle.

Einfache Updates, echte Anpassbarkeit

Ein selbst gehostetes System zu aktualisieren sollte kein eigenes Projekt sein. Bei MyCompany bedeutet ein Update, eine einzige JAR-Datei im Klassenpfad des Servers zu ersetzen — die veröffentlichten Versionen liegen auf download.lsfusion.org. Und weil die Anwendung aus lsFusion-Modulen besteht, erweitern Sie sie auf Ihrem eigenen Server, ohne irgendetwas neu zu kompilieren: Legen Sie eine .lsf-Datei neben die Anwendung und verweisen Sie logics.topModule darauf.

FAQ

Was sind die Serveranforderungen?

Eine Maschine mit mindestens 4 GB RAM unter Ubuntu 18+, Debian 9+, RHEL 8+, CentOS 8+, Fedora 35+ oder Windows. Der Web-Client läuft auf Port 8080, der Desktop-Client auf Port 7652.

Wie lange dauert die Installation?

Etwa 10-15 Minuten auf dem eigenen Server. Unter Linux ist es ein einziger Befehl; unter Windows starten Sie einen eigenständigen Installer. Danach öffnen Sie einfach den Web-Client unter der Adresse Ihres Servers auf Port 8080.

Kann ich es in Docker betreiben?

Ja. Das Docker-Deployment ist offiziell dokumentiert — Sie können MyCompany neben Ihren anderen containerisierten Diensten betreiben.

Wie funktionieren Updates?

Sie ersetzen eine einzige JAR-Datei im Klassenpfad des Servers durch eine neuere Version von download.lsfusion.org. Keine Lizenz-Reaktivierung, denn es gibt keine Lizenzen zu aktivieren.

Funktioniert es im lokalen Netzwerk, ohne Cloud?

Ja. Alles läuft auf Ihrem eigenen Server — das System funktioniert in Ihrem LAN ohne Abhängigkeit von einem externen Cloud-Dienst; die Mitarbeiter öffnen einfach den Web-Client im Browser.

Sind meine Daten eingesperrt?

Nein. Ihre Daten liegen in einer Standard-PostgreSQL-Datenbank auf Ihrem eigenen Server, und der Anwendungscode ist Open Source unter Apache 2.0. Sie können die Daten jederzeit direkt abfragen, sichern oder migrieren.
