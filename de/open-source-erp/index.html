Open-Source-ERP-Software zum Selbsthosten und Anpassen

MyCompany ist ein Open-Source-ERP für kleine und mittlere Unternehmen. Der vollständige Quellcode ist unter der Apache-2.0-Lizenz auf GitHub veröffentlicht — Sie können das System auf Ihrem eigenen Server betreiben, seine Funktionsweise studieren und alles ändern, was Sie möchten: ohne Lizenzgebühren, ohne versteckte Kosten und ohne Anbieterbindung.

Die meisten Unternehmen, die ein ERP für KMU evaluieren, stehen vor demselben Dilemma: ein bezahlbares Produkt von der Stange, das ihnen seine Arbeitsweise aufzwingt, oder eine maßgeschneiderte Entwicklung, die ein Vermögen kostet. MyCompany ist für Nicht-Standard-Prozesse gemacht. Sie starten mit fertigen Modulen für die zentralen Geschäftsprozesse und passen anschließend Datenmodell, Formulare und Logik in der Sprache lsFusion an, bis das System der tatsächlichen Arbeitsweise Ihres Unternehmens entspricht.

Was Sie von Anfang an bekommen

MyCompany wird als vollständiges ERP ausgeliefert, nicht als leeres Framework. Das Standardpaket umfasst:

  • Stammdaten und Administration
  • Buchhaltung: Kontenplan, Journale, Anlagevermögen
  • Lager und Rechnungsstellung
  • Einkauf und Verkauf
  • Produktion
  • Einzelhandel und Kasse
  • CRM und Projekte
  • Personal und Transport

Wirklich Open Source: Apache 2.0

Das gesamte Produkt liegt in einem öffentlichen Repository unter der Apache-2.0-Lizenz — einer der freizügigsten Open-Source-Lizenzen überhaupt. Sie dürfen MyCompany kommerziell nutzen, verändern und Ihre Änderungen weiterverbreiten, ohne jemanden zu bezahlen oder um Erlaubnis zu fragen. Es gibt keine „Community Edition“, bei der die nützlichen Teile fehlen: Was Sie auf GitHub sehen, ist das komplette System.

Ein anpassbares ERP — ohne den Code zu forken

Die Anpassbarkeit unterscheidet MyCompany von den meisten Open-Source-ERP-Systemen. Sie patchen nicht den Quellcode des Produkts und pflegen keinen Fork. Stattdessen fügen Sie Ihr eigenes lsFusion-Modul hinzu — eine einfache .lsf-Textdatei, die mit REQUIRE MyCompany beginnt —, legen sie auf den Server und verweisen die Einstellung logics.topModule darauf. Ihr Modul erweitert oder überschreibt Datenmodell, Formulare und Standardlogik, während das Produkt selbst unangetastet bleibt — es muss nichts neu kompiliert werden. Der komplette Entwicklungsworkflow mit einer IDE auf Basis von IntelliJ IDEA und der lsFusion-Plattform ist in der Dokumentation beschrieben.

Ein solides technisches Fundament

MyCompany basiert auf lsFusion, einer Open-Source-Low-Code-Plattform (LGPL v3), die auf Java und PostgreSQL läuft und ACID-Garantien von Haus aus mitbringt. Nach Angaben des Plattformherstellers skaliert lsFusion auf Tausende gleichzeitige Benutzer und Terabytes an Daten — ein System, das Sie als kleines Unternehmen einführen, muss also nicht ersetzt werden, wenn Sie wachsen.

In Minuten selbst gehostet, Updates mit einer Datei

Sie können MyCompany in 10-15 Minuten auf Ihrem eigenen Server installieren: ein Einzeiler-Skript unter Linux, ein Installer unter Windows oder Docker. Eine Maschine mit 4 GB RAM reicht für den Start, und die Benutzer arbeiten über einen Web-Client auf Port 8080. Ein Systemupdate bedeutet, eine einzige JAR-Datei zu ersetzen. Die Dokumentation ist auf Englisch, Russisch und Polnisch verfügbar, und eine Live-Demo lässt Sie das System erkunden, bevor Sie irgendetwas installieren.

FAQ

Unter welcher Lizenz wird MyCompany vertrieben?

MyCompany steht unter Apache 2.0, der vollständige Quellcode ist öffentlich auf GitHub. Die zugrunde liegende lsFusion-Plattform ist ebenfalls Open Source, unter LGPL v3.

Darf ich es kostenlos kommerziell nutzen?

Ja. Die Apache-2.0-Lizenz erlaubt kommerzielle Nutzung, Veränderung und Weiterverbreitung ohne jede Gebühr. Keine Lizenzkosten, keine versteckten Gebühren.

Wie passe ich MyCompany an?

Sie schreiben Ihr eigenes lsFusion-Modul — eine .lsf-Datei mit REQUIRE MyCompany —, legen es auf den Server und setzen logics.topModule auf seinen Namen. Änderungen an Datenmodell, Formularen und Logik leben in Ihrem Modul — Produktupdates überschreiben Ihre Arbeit nicht, und nichts muss neu kompiliert werden.

Welche Module sind enthalten?

Stammdaten, Buchhaltung, Lager, Rechnungsstellung, Einkauf, Produktion, Verkauf, Einzelhandel/Kasse, CRM, Projekte, Personal, Transport und Administration.

Gibt es eine Anbieterbindung?

Nein. Sie hosten das System selbst, der Code ist offen, und Ihre Daten liegen in einer Standard-PostgreSQL-Datenbank auf Ihrer eigenen Hardware. Sollten Sie unsere Leistungen eines Tages nicht mehr nutzen, bleiben Software und Daten vollständig Ihre.

Wie schneidet es im Vergleich zu rein gehosteten ERP-Systemen ab?

Vom Anbieter gehostete ERP-Produkte können bequem sein, aber der Anbieter kontrolliert Ihre Daten, die Preise und den Update-Zeitplan. Mit MyCompany tauschen Sie einen kleinen Einrichtungsaufwand — 10-15 Minuten auf Ihrem eigenen Server — gegen die volle Kontrolle darüber, wo das System läuft und wie es sich weiterentwickelt.
