ERP für Entwickler · alles als Code

Open-Source-ERP-Plattform für Entwickler

MyCompany ist ein produktionsreifes ERP auf lsFusion für Teams, die echte Geschäftslogik verändern müssen. Definieren Sie Datenmodell, Berechnungen, Constraints, Aktionen und Formulare in einer deklarativen Sprache, versionieren Sie alles mit Git und erweitern Sie das Produkt ohne eigenen Fork.

Geschäftssysteme statt Framework-Infrastruktur

Ein Modell von der Datenbank bis zur Oberfläche

lsFusion hält die wichtigen ERP-Bausteine in einer konsistenten Codebasis. Persistenz, Transaktionen und UI-Verhalten entstehen aus deklarativen Definitionen; die Anwendung bleibt normaler Text für Review, Branches und Deployment.

LSF

Eine deklarative Sprache

Beschreiben Sie gespeicherte und berechnete Daten, Constraints, Ereignisse, Aktionen und Formulare, ohne Regeln auf ORM, Services und Frontend zu verteilen.

IDE

Echte Entwicklerwerkzeuge

Arbeiten Sie in einer IntelliJ-IDEA-basierten Umgebung mit Vervollständigung, Fehleranzeige, Verwendungssuche und Breakpoints bei Datenänderungen.

MOD

Module statt Forks

Legen Sie Firmenlogik in ein separates .lsf-Modul mit REQUIRE MyCompany. Beim Update wird die Produkt-JAR ersetzt, die Erweiterung bleibt.

Ein verständlicher Stack

Vom .lsf-Quellcode zum Drei-Schichten-ERP

Der lsFusion-Server läuft auf der JVM, speichert Daten in PostgreSQL und liefert einen Webclient. Deklarative Logik läuft nahe an den Daten, jede Operation ist transaktional und das Modul erweitert alle Schichten.

Entwicklungsleitfaden

Wie sieht der Technologie-Stack aus?

Der Server läuft auf der JVM, Daten liegen in PostgreSQL und Benutzer arbeiten im Webclient. Logik und Formulare werden in lsFusion geschrieben.

Wie richte ich die Entwicklungsumgebung ein?

Installieren Sie IntelliJ IDEA mit lsFusion-Plugin, klonen Sie MyCompany und folgen Sie dem Leitfaden zum lokalen Start von Anwendung und Modul.

Wie überstehen Module Produkt-Updates?

Das .lsf-Modul benötigt MyCompany als Abhängigkeit, statt es zu patchen. Das Update ersetzt die Produkt-JAR und behält Ihr Modul.

Gibt es eine API?

Ja. MyCompany unterstützt HTTP, JSON, XML, CSV, XLS und DBF; externe Systeme benötigen keinen direkten Datenbankzugriff.

Kann ich kommerzielle Lösungen bauen?

Ja. MyCompany nutzt Apache 2.0, die Plattform LGPL v3. Es gibt keine Lizenzschlüssel und kein DRM.

Kann ich die Sprache ohne Installation testen?

Ja. Nutzen Sie die Online-Umgebung, prüfen Sie GitHub oder installieren Sie MyCompany auf Ihrem Server.
