Бесплатная программа складского учёта с открытым кодом

MyCompany — бесплатная программа складского учёта с открытым исходным кодом под лицензией Apache 2.0. Она поддерживает любое количество складов, зоны и ячейки хранения, приёмки и отгрузки, внутренние перемещения, партии и серийные номера, штрихкоды, мобильную комплектацию и инвентаризацию. Никаких скрытых платежей и привязки к поставщику: код открыт, а данные всегда остаются у вас.

Программа устанавливается на собственном сервере за 10-15 минут и работает через браузер. В основе — открытая платформа lsFusion (LGPL v3) на Java и PostgreSQL: складские операции выполняются в ACID-транзакциях, поэтому остатки остаются корректными даже при одновременной работе многих пользователей.

При этом склад — не изолированная программа, а часть полноценной ERP-системы: закупки, продажи, производство и расчёты работают с теми же товарами и остатками в единой базе.

Склады, зоны и ячейки

Места хранения организованы в иерархическое дерево любой глубины: склад делится на зоны, зоны — на ячейки. Число складов не ограничено, а ячейки необязательны — небольшой склад можно вести без адресного хранения и включить его позже. Для локаций настраиваются запрет отрицательных остатков и лимит резервов, причём настройки наследуются вниз по дереву.

Приёмки, отгрузки и перемещения

Приёмки проходят статусы «Черновик» → «Готов» → «Выполнен» и поддерживают адресное размещение по ячейкам. Подтверждение заказа поставщику автоматически создаёт приёмку, начальные остатки импортируются из Excel, а текущие — выгружаются обратно. Отгрузки контролируют доступность товара и резервируют его под заказ. Перемещения между складами выполняются с двусторонним подтверждением: принимающий склад получает товар по той же себестоимости, что и на исходном складе. Порча, потери, брак и просрочка оформляются документами списания, а расхождения остатков — инвентаризациями.

Штрихкоды, партии и серийные номера

Партионный учёт включается глобально, на категорию или на отдельный товар. Идентификаторы партий генерируются автоматически по шаблону «префикс + счётчик» (например, LOT-000123), а для штучного отслеживания есть режим серийных номеров. Этикетки со штрихкодами печатаются в PDF, DOCX, XLSX, RTF и HTML; если при сканировании встретился неизвестный штрихкод, программа может тут же создать по нему партию. Несколько товаров продажи могут учитываться как одна складская позиция (SKU) с коэффициентом пересчёта, а упаковки настраиваются по типам документов.

Мобильная комплектация

Для отгрузок в статусе «Готов» задания на отбор создаются автоматически. Кладовщик выполняет их в мобильной форме прямо в браузере — сканирует штрихкоды ячеек и товаров, не устанавливая отдельных приложений. Так же с мобильного устройства выполняются и перемещения.

Инвентаризация

Инвентаризационная ведомость заполняется по локации, по категории или вручную со сканированием штрихкодов. Программа сама рассчитывает излишки и недостачи и формирует корректировки остатков. Большой склад можно пересчитывать несколькими командами по отдельным ведомостям — итоги суммируются автоматически, а ведомости и этикетки печатаются из программы.

Отчёты и доступ по API

Отчёт по остаткам показывает величины «Ожидается», «Резерв», «Доступно» и «На складе», срез на любую дату, сводные таблицы (pivot), историю движения и разбивку по партиям. Доступны стоимостная оценка запасов и отчёт по себестоимости — учёт ведётся на трёх регистрах: остатков, резервов и себестоимости. Текущие остатки можно получать и по HTTP API в формате JSON с фильтрами по категории и локации.

Частые вопросы

Программа действительно бесплатная?

Да. MyCompany распространяется бесплатно под лицензией Apache 2.0 — без скрытых платежей и привязки к поставщику. Система работает на вашем собственном сервере, поэтому данные принадлежат только вам.

Сколько складов можно вести?

Сколько угодно. Каждый склад описывается деревом локаций любой глубины — от одной общей площадки до зон и ячеек. Адресное хранение можно включить только там, где оно действительно нужно.

Есть ли поддержка штрихкодов и терминалов сбора данных?

Да. Комплектация, перемещения и заполнение инвентаризаций выполняются со сканированием штрихкодов ячеек и товаров в мобильной форме, которая открывается в обычном браузере — устанавливать отдельное приложение не нужно.

Поддерживаются ли партии и серийные номера?

Да. Партионный учёт включается глобально, для категории или конкретного товара, идентификаторы партий генерируются автоматически. Для поштучного отслеживания предусмотрен режим серийных номеров, а этикетки со штрихкодами печатаются прямо из программы.

Как провести инвентаризацию?

Создайте документ инвентаризации и заполните его по локации, по категории или вручную со сканированием. Программа автоматически рассчитает излишки и недостачи и скорректирует остатки. Большой склад удобно считать несколькими командами по отдельным ведомостям — итоги сложатся сами.

Можно ли получить остатки по API?

Да. Текущие остатки доступны по HTTP API в формате JSON с фильтрами по категории товара и локации.
