ERP с открытым исходным кодом для вашего бизнеса

MyCompany — это бесплатная ERP с открытым исходным кодом. Весь код системы опубликован на GitHub под лицензией Apache 2.0: её можно свободно использовать в коммерческих целях, изменять и дорабатывать под собственные процессы. Никаких лицензионных ключей, скрытых платежей и обязательных подписок.

Система устанавливается на собственном сервере за 10-15 минут и уже в базовой комплектации закрывает основные контуры управления предприятием: справочники, бухгалтерию, склад, счета, закупки, производство, продажи, POS, CRM, проекты, кадры и транспорт.

В основе MyCompany — открытая low-code платформа lsFusion (лицензия LGPL v3) на Java и PostgreSQL с поддержкой ACID-транзакций из коробки. Благодаря этому open source ERP остаётся по-настоящему расширяемой: недостающую логику можно дописать самостоятельно, не дожидаясь вендора.

Что даёт лицензия Apache 2.0

Исходный код MyCompany размещён в репозитории github.com/lsfusion-solutions/mycompany, условия использования зафиксированы в файле LICENSE. Apache 2.0 — одна из самых свободных open-source лицензий: она разрешает коммерческое использование, модификацию и распространение изменённых версий. В системе нет лицензионных ключей и DRM — вы контролируете и код, и данные.

Никакой привязки к поставщику

ERP с открытым кодом избавляет от главного риска проприетарных систем — зависимости от вендора. MyCompany работает на вашем сервере, все данные хранятся в PostgreSQL, а средствами платформы их можно выгружать в популярные форматы: JSON, XML, CSV, XLS и DBF. Обновление сводится к замене JAR-файла, и вы сами решаете, когда и на какую версию переходить.

Доработка без перекомпиляции ядра

Чтобы изменить или расширить логику, не нужно пересобирать систему. Вы создаёте собственный модуль на языке lsFusion — файл .lsf, который начинается с директивы REQUIRE MyCompany, — кладёте его на сервер и указываете в параметре logics.topModule. Стандартный код при этом остаётся нетронутым, поэтому ваши доработки переживают обновления. Полный цикл разработки — с IntelliJ IDEA и платформой lsFusion — подробно задокументирован.

Проверенные открытые технологии

Платформа lsFusion сама распространяется как open source (LGPL v3) и построена на Java и PostgreSQL с гарантиями ACID из коробки; для разработчиков есть IDE на базе IntelliJ IDEA. По заявлению разработчика платформы, она рассчитана на тысячи одновременных пользователей и терабайты данных, так что системе есть куда расти вместе с бизнесом.

Быстрый старт

Установить MyCompany можно скриптом одной командой, установщиком для Windows или через Docker — достаточно сервера с 4 ГБ ОЗУ. Документация доступна на русском, английском и польском языках. А чтобы оценить систему без установки, откройте живое демо.

Вопросы и ответы

Под какой лицензией распространяется MyCompany?

Под лицензией Apache 2.0 — текст лежит в файле LICENSE в репозитории на GitHub. Она разрешает бесплатное коммерческое использование, изменение кода и распространение собственных версий системы.

Чем ERP с открытым кодом полезна бизнесу?

Открытый код означает отсутствие привязки к поставщику: систему можно проверить (аудит кода), доработать своими силами или силами любого подрядчика и развернуть на собственной инфраструктуре. Данные всегда остаются под вашим контролем в PostgreSQL.

Как вносить доработки и обновляться без потери изменений?

Доработки оформляются отдельным модулем .lsf с директивой REQUIRE MyCompany, который подключается через параметр logics.topModule. Ядро системы не изменяется, поэтому при обновлении достаточно заменить JAR-файл — ваши модули продолжат работать.

На чём написана система?

MyCompany построена на открытой платформе lsFusion (LGPL v3): сервер на Java, хранение данных в PostgreSQL, транзакции ACID из коробки. Логика описывается на языке lsFusion, разработка ведётся в IDE на базе IntelliJ IDEA.

Есть ли сообщество и каналы поддержки?

Да. Вопросы по разработке и использованию можно задать в открытых каналах сообщества — Telegram и Slack, а для гарантированной поддержки доступны платные услуги команды разработчиков.
